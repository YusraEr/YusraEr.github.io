Perjalanan kita dalam dunia *software testing* semakin mendalam. Kita sudah membahas *Testing Plan* (strategi), *UI/UX Testing* (apa yang dilihat pengguna), dan *Unit Testing* (fondasi kode). Sekarang, materi dari Kelompok 6 membawa kita ke **API Testing**.

Jika aplikasi adalah sebuah restoran, UI adalah ruang makan dan pelayannya, maka API (*Application Programming Interface*) adalah dapurnya. Di sinilah semua permintaan diproses, data diambil, dan pesanan disiapkan sebelum diantar kembali ke pelanggan.

### Anatomi Request & Response

Dalam API testing, kita pada dasarnya mengirim "pesanan" (Request) ke "dapur" (Server) dan memeriksa "hidangan" (Response) yang kembali.

<Row fillWidth mobileDirection="column" gap="16" marginBottom="24" marginTop="16">
  <Column fillWidth border="neutral-alpha-medium" padding="16" radius="m" gap="8">
    <Text variant="heading-strong-s">Anatomi Request (Klien ke Server)</Text>
    <Text variant="body-default-s" onBackground="neutral-weak" marginBottom="8">
      Ini adalah permintaan yang kita kirim. Komponen utamanya adalah:
    </Text>
    <CodeBlock
        radius="m"
        codes={[
          {
            code:
`GET /api/v1/users/123
Host: api.example.com
Authorization: Bearer <token>`,
            language: "http",
            label: "Contoh HTTP Request"
          }
        ]}
    />
    <Text variant="body-default-s" onBackground="neutral-weak" marginTop="8">
      • **Method (HTTP Verb)**: Aksi yang ingin dilakukan (`GET` untuk mengambil data, `POST` untuk mengirim data baru).
    </Text>
    <Text variant="body-default-s" onBackground="neutral-weak">
      • **URL (Endpoint)**: Alamat dari sumber daya yang ingin diakses (`/api/v1/users/123`).
    </Text>
  </Column>

  <Column fillWidth border="neutral-alpha-medium" padding="16" radius="m" gap="8">
    <Text variant="heading-strong-s">Anatomi Response (Server ke Klien)</Text>
    <Text variant="body-default-s" onBackground="neutral-weak" marginBottom="8">
      Ini adalah balasan dari server setelah memproses *request*.
    </Text>
    <CodeBlock
        radius="m"
        codes={[
          {
            code:
`Status: 200 OK
Content-Type: application/json

{
  "id": 123,
  "name": "Mahasiswa Keren"
}`,
            language: "http",
            label: "Contoh HTTP Response"
          }
        ]}
    />
    <Text variant="body-default-s" onBackground="neutral-weak" marginTop="8">
      • **Status Code**: Kode hasil eksekusi. Ini sangat penting! (`200 OK` berarti sukses, `404 Not Found` berarti data tidak ada, `401 Unauthorized` berarti tidak punya izin).
    </Text>
  </Column>
</Row>


### "Senjata" untuk API Testing

Kita tidak melakukan ini secara manual. Materi ini menyoroti dua alat utama:

<Row fillWidth mobileDirection="column" gap="16" marginBottom="24">
  <Column fillWidth border="neutral-alpha-medium" padding="16" radius="m" gap="8">
    <Text variant="heading-strong-s">Postman</Text>
    <Text variant="body-default-s" onBackground="neutral-weak">
      Ini adalah pilihan paling populer saat ini. Sangat *user-friendly*, bagus untuk testing REST API, mendukung manajemen *collections* (kumpulan *request*), *environment variables* (untuk membedakan *testing* vs *production*), dan *testing* otomatis.
    </Text>
  </Column>
  <Column fillWidth border="neutral-alpha-medium" padding="16" radius="m" gap="8">
    <Text variant="heading-strong-s">SoapUI</Text>
    <Text variant="body-default-s" onBackground="neutral-weak">
      Ini adalah alat yang lebih "berat" dan sangat kuat, terutama untuk API berbasis SOAP (yang menggunakan XML). Ini sangat cocok untuk skenario *enterprise* yang kompleks dan mendukung pengujian tingkat lanjut seperti *security* dan *load testing*.
    </Text>
  </Column>
</Row>
