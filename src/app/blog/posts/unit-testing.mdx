Di *post* blog sebelumnya, kita telah menjelajahi dunia *testing* dari level strategis (`Test Plan`), level pengguna (`UI/UX Testing`), hingga level taktikal (`Test Case`). Sekarang, kita tiba di fondasi dari semua piramida kualitas. Materi dari Kelompok 5 kali ini adalah tentang **Unit Testing**.

Ini adalah "jaring pengaman" bagi kami, para *developer*.

### Apa Sebenarnya Unit Testing Itu?

<Feedback
    icon
    variant="info"
    title="Apa Itu Unit Testing?"

    marginBottom="24"
/>

Materi ini menjelaskannya dengan **Piramida Pengujian**. *End-to-End Testing* (E2E) ada di puncak: lambat dan mahal. *Integration Testing* ada di tengah. Dan **Unit Testing** ada di dasar: cepat, murah, dan seharusnya menjadi yang paling banyak jumlahnya.

Anologi terbaiknya? Saat merakit mobil, kita tidak langsung menguji mobilnya. Kita uji dulu mesinnya, remnya, lampunya, secara terpisah. Itulah *unit testing*.

### Pola Emas: Arrange, Act, Assert (AAA)

Bagaimana cara menulis *unit test* yang baik? Ada pola universal yang disebut **AAA (Arrange, Act, Assert)**. Ini membuat tes kita bersih, terstruktur, dan mudah dibaca.

<Grid fillWidth gap="16" columns="3" mobileColumns="1" marginBottom="24">
  <Column fillWidth border="neutral-alpha-medium" padding="16" radius="m" gap="8">
    <Text variant="heading-strong-s">1. Arrange (Persiapan)</Text>
    <Text variant="body-default-s" onBackground="neutral-weak">
      Siapkan semua kondisi awal dan *input* yang diperlukan untuk tes. Misalnya, membuat objek, mengisi variabel, atau *mocking* dependensi.
    </Text>
  </Column>
  <Column fillWidth border="neutral-alpha-medium" padding="16" radius="m" gap="8">
    <Text variant="heading-strong-s">2. Act (Aksi)</Text>
    <Text variant="body-default-s" onBackground="neutral-weak">
      Panggil atau jalankan satu fungsi/metode yang ingin kita uji. Ini adalah inti dari tes, dan idealnya hanya satu baris kode.
    </Text>
  </Column>
  <Column fillWidth border="neutral-alpha-medium" padding="16" radius="m" gap="8">
    <Text variant="heading-strong-s">3. Assert (Penegasan)</Text>
    <Text variant="body-default-s" onBackground="neutral-weak">
      Verifikasi bahwa hasil dari "Act" sesuai dengan yang kita harapkan. Apakah *output*-nya benar? Apakah statusnya berubah? Jika *assert* gagal, tes akan gagal.
    </Text>
  </Column>
</Grid>

### Contoh "Live Coding" (Versi Pytest)

Materi ini juga menyertakan contoh kode. Mari kita lihat contoh `ShoppingCart` menggunakan **Pytest** dan pola AAA.

<CodeBlock
    marginBottom="24"
    codes={[
      {
        code:
`# File: shopping_cart.py
from typing import List

class ShoppingCart:
    def __init__(self, max_size: int) -> None:
        self.items: List[str] = []
        self.max_size = max_size

    def add(self, item: str):
        if len(self.items) >= self.max_size:
            raise OverflowError("cannot add more items")
        self.items.append(item)

    def get_items(self) -> List[str]:
        return self.items
`,
        language: "python",
        label: "Kode Aplikasi (shopping_cart.py)"
      },
      {
        code:
`# File: test_shopping_cart.py
import pytest
from shopping_cart import ShoppingCart

def test_add_item():
    # 1. Arrange
    cart = ShoppingCart(5)
    
    # 2. Act
    cart.add("apple")
    
    # 3. Assert
    assert len(cart.get_items()) == 1
    assert "apple" in cart.get_items()

def test_add_overflow_error():
    # 1. Arrange
    cart = ShoppingCart(1)
    cart.add("apple")
    
    # 2. Act & 3. Assert (digabung untuk mengecek error)
    with pytest.raises(OverflowError):
        cart.add("banana")
`,
